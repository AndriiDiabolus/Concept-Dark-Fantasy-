## Правила для Claude

### Язык
- Общение с пользователем — **только на русском**
- Документация проекта — на русском (технические термины допускаются на английском)

### Структура файлов
- `CLAUDE.md` — только правила и инструкции для Claude. **Не хранить здесь контент проекта.**
- `context.md` — весь контекст проекта: концепция, сеттинг, механики, история изменений, решения
- При добавлении нового контента → обновлять `context.md`, не `CLAUDE.md`

### Работа с контекстом
- Перед ответом на вопросы о проекте — читать `context.md`
- При каждом значимом решении (механика, нарратив, архитектура) — дописывать в `context.md` в раздел "История изменений"
- Изменения фиксировать с датой: `### YYYY-MM-DD — Описание`

### Стиль ответов
- Коротко и по делу
- Предлагать варианты, а не одно решение (кроме случаев когда вопрос технический и ответ однозначный)
- "Няма" от пользователя = всё верно, продолжать в том же духе

### Git
- Коммитить только когда пользователь явно просит
- Не пушить без подтверждения
- Формат коммита: `тип: краткое описание` (feat, docs, design, fix)

### Приоритеты проекта
1. Концепция и дизайн-документ — сначала
2. Механики на бумаге — потом
3. Прототип — после утверждения концепции
