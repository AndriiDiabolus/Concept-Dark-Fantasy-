## Правила для Claude

### Язык
- Общение с пользователем — **только на русском**
- Документация проекта — на русском (технические термины допускаются на английском)

### Структура файлов
- `CLAUDE.md` — только правила и инструкции для Claude. **Не хранить здесь контент проекта.**
- `context.md` — весь контекст проекта: концепция, сеттинг, механики, история изменений, решения
- При добавлении нового контента → обновлять `context.md`, не `CLAUDE.md`

### Работа с контекстом
- Перед ответом на вопросы о проекте — читать `context.md`
- При каждом значимом решении (механика, нарратив, архитектура) — дописывать в `context.md` в раздел "История изменений"
- Изменения фиксировать с датой: `### YYYY-MM-DD — Описание`

### Стиль ответов
- Коротко и по делу
- Предлагать варианты, а не одно решение (кроме случаев когда вопрос технический и ответ однозначный)
- "Няма" от пользователя = всё верно, продолжать в том же духе

### Git
- Коммитить только когда пользователь явно просит
- Не пушить без подтверждения
- Формат коммита: `тип: краткое описание` (feat, docs, design, fix)

### Приоритеты проекта
1. Концепция и дизайн-документ — сначала
2. Механики на бумаге — потом
3. Прототип — после утверждения концепции

### Сессия 2 — 2026-03-16 (часть 1)
**Сделано:** Полная детализация структуры уровней, механик и нарратива Act I.
- **4 уровня Act I прописаны:** Січ → Деревни → Подход к замку → Цитадель (4 суб-локации)
- **Механика нечестивости детализирована:** 3 уровня + шкала демона + энрейдж с эволюцией (наказание → инструмент → ловушка)
- **Визуальная деградация:** фиолетовая рука → контур+глаза → метаморфоза (копыта/рога/хвост)
- **5 точек невозврата:** четкая структура падения героя
- **Семья:** пленники, транспортировка, мотанка уронена дочерью, открытый финал
- **Художественная глава** полностью написана (`chapter_1_draft.md` + `.txt` версия)
- **Риск энрейджа определён:** 4 концептуальных подхода предложено (глитчи управления + психо-фидбек + контроль ресурса)

**Мнение Claude:** Концепция исключительно сильна (уровень Hades/Spec Ops). Главный риск — баланс энрейджа.

### Сессия 2 — 2026-03-16 (часть 2 — финал)
**Сделано:** Решены все 4 блокирующих риска. Полная детализация механик и интеграция всех спецификаций.

**Решены блокеры:**
1. ✅ **Баланс энрейджа:** уязвимость снижена до 2 сек, фаза 4 неконтролируемая, иллюзия выбора честная
2. ✅ **Пейсинг:** формализовано 40–45 мин (5–7 мин/уровень + 3 ролика)
3. ✅ **Босс Князя:** 4-фазный файт (охрана → копьё → охрана → дуэль) с уникальными механиками
4. ✅ **Способность Одержимость:** точные числа (20 сек, 2 мин CD, когти, рывок, лазание, 2 сек уязвимость)

**Детализация механик:**
- Шкала нечестивости заполняется от **атак И урона** (не только атак)
- Способность "Одержимость" открывается на уровне 3, заменяет меч на когти, даёт рывок и лазание по стенам
- Энрейдж эволюция: неконтролируемый L1-2 → управляемый L3 → неконтролируемый в Фазе 4 Князя
- Фаза 4 Князя: критический момент когда козак добровольно нажимает "Одержимость" → сразу же теряет контроль
- Это архитектурно честно: игрок прошёл весь путь, заполнил шкалу, научился управлять, но в финале уже не может остановиться
